# 总览

这是一个地图探索+卡牌战斗游戏。
玩家有自己的集换式卡组，可以在地图探索时遇见怪物与之战斗，也可能遇见一些宝箱、商人等特殊事件来获得奖励。
奖励可能包含：新的卡牌加入卡组、从卡组中删除卡牌、金币（用于和商人交易）、道具（永久发挥作用，类似杀戮尖塔中的遗物）
玩家需要探索地图，构筑卡组，获得道具，打败或者净化敌人，最终击败boss。
你可以在cards.csv中看到目前的所有卡牌的列表
你可以在items.csv中看到目前的所有道具的列表

# 地图探索规则

玩家进入游戏后，默认进入地图探索界面。
玩家在地图探索时，视角是2.5d俯视角，类似星露谷物语。按下WASD进行移动，按下E与场景中的物体或者NPC进行交互
地图由一个个“房间”组成，房间按照功能可以划分为以下几类：
+ 怪物房，房间中有一个怪物NPC，玩家与之互动之后进入与它的战斗画面，战斗结束之后再次回到地图探索界面。
+ 商店，房间中有商人NPC，与之互动能够购买卡牌或者道具（交易的货币是金币）。
+ 宝箱房，奖励金币、卡牌或者道具
+ （等待构思）

其中，商店会列出5张随机卡牌、5个道具待购买（一般来说道具比卡牌贵），也可以花钱删除卡组中的卡牌，删除的次数不限但是每次需要消耗的金币变多，（5、10、15、20这样变化）

宝箱互动打开之后会有一些金币、卡牌二选一和一个道具。

当玩家身处一个房间的时候，他的整个屏幕就是这个房间中的内容，相机不会跟随移动，玩家也不会看到整个地图的模样（类似以撒的结合），当玩家完成这个房间中的事情之后（一般来说就是和怪物战斗结束之后，如果是商店宝箱等就没有这个限制），就可以从出口离开房间，出口可能不止一个，玩家可以从出口处看到通向何种种类的房间，离开房间后就可以进入对应的房间。
所以房间就是整个游戏地图的单元，同时，房间之间的连接关系不是提前定义的，而是随机生成的，但是也不是完全随机，会保证一定的约束关系，例如从开始到boss中间一共多少房间，其中经过多少怪物房，多少商店房间等，但是具体顺序是随机的，这一点类似哈迪斯。
在开发时，可以把每个房间作为一个单元分别进行开发。

# 卡牌战斗规则

牌有这些属性：名称，消耗时间，种类，效果；其中种类分为攻击、防御和净化。

怪物有这些属性：食物块列表（类似于血量，但是不同的是食物块有自己的特性）、净化条件。
其中食物块列表类似一个队列，玩家每次吃（也就是进攻）都是从最前端开始按顺序来吃，
每种食物块至少包含需要消化的时间这个属性，也可能包含其他特殊效果。
净化条件是一系列需要完成的“任务”，通常由对应的卡牌进行净化，也可能包含特殊净化条件或者限制条件。
最基础的净化条件有这五种：脏污、腐败、劣质、虫蚀、污秽，都由对应的净化卡牌来净化。
玩家可以按照任意顺序进行净化。
当怪物的食物块被吃完*或者*所有净化条件完成之后，玩家战斗胜利。
当玩家在战斗中生命值归0时，玩家战斗失败。

玩家有生命值，初始为20，且会跨战斗保存。
玩家吃下的食物块会以队列的形式存储在“胃”中，胃的容量是有限的，初始为3.
如果胃中的剩余容量不够，则无法打出对应的吃的卡牌（例如吃2，但是剩余容量只有1.）
每过一个单位时间，胃中最前面的食物（也就是最先吃的食物）的消化所需时间减少1（也称为消化1），减少到0或以下时则弹出队列。
玩家战斗开始时的防御是0，玩家可以通过卡牌给自己添加防御，当玩家受到敌人的攻击的时候，如果玩家的防御大于0，则先尽可能扣除防御点数，然后将剩余的部分从玩家的生命值中扣除。

游戏有“时间”的概念，记录全局时间，初始时t=0.
一般来说打出卡牌或者使用道具等需要消耗时间。
敌人的行动会按照一定规则分布在时间轴上，一个简单的例子是每5个单位的时间发动一次攻击力为2的攻击。
玩家能看到时间轴上的一个窗口，宽10个单位，起点是当前的时间，也可以看到时间轴上显示的接下来怪物的行动意图。

战斗开始时，玩家携带的卡牌全部进入抽牌堆，玩家抽4张牌。
玩家可以选择一张手牌打出，打出后，游戏时间向后推进该卡牌消耗的时间，推进时间时，先消化胃中最前面的食物，然后如果其中遇到了怪物行动的时间点，则怪物行动，行动完成后再结算玩家的打出的卡牌。如果恰好在推进卡牌消耗的时间后怪物有行动需要执行，也是怪物的行动优先，然后执行玩家的行动。

玩家打出的卡牌效果结算成功后，那张牌进入弃牌堆，然后立即从抽牌堆抽一张牌（不消耗时间），如果抽牌堆是空的，则把弃牌堆中的牌打乱后全部洗回抽牌堆，如果弃牌堆也没有牌，那么不进行抽牌。
